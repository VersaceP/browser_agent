# 修复记录 (2026-04-15)

## 问题 1: 错误的 API 调用方式

### 问题描述
初始实现错误地使用了 Anthropic Messages API 格式来调用 MiniMax，但用户实际使用的是 **MiniMax Coding Plan API**。

### 错误代码
```python
# ❌ 错误：使用 Anthropic SDK
import anthropic

message = self.client.messages.create(
    model="MiniMax-M2.7",
    max_tokens=2000,
    messages=[{
        "role": "user",
        "content": [
            {"type": "image", "source": {"type": "base64", "data": image_data}},
            {"type": "text", "text": prompt}
        ]
    }]
)
```

### 修复方案
```python
# ✅ 正确：使用 MiniMax Coding Plan API
import requests

url = f"{api_host}/v1/tools/understand_image"
headers = {"Authorization": f"Bearer {api_key}"}
payload = {
    "prompt": prompt,
    "image_source": abs_image_path  # 使用绝对路径，不是 base64
}

response = requests.post(
    url,
    headers=headers,
    json=payload,
    proxies={"http": "http://127.0.0.1:7890", "https": "http://127.0.0.1:7890"},
    timeout=30
)
```

### 关键差异

| 特性 | Anthropic API | MiniMax Coding Plan API |
|------|---------------|-------------------------|
| SDK | `anthropic` | `requests` |
| 图片格式 | base64 编码 | 文件绝对路径 |
| 环境变量 | `ANTHROPIC_AUTH_TOKEN` | `MINIMAX_API_KEY` |
| API 端点 | `/v1/messages` | `/v1/tools/understand_image` |
| 代理 | 不需要 | 需要（http://127.0.0.1:7890） |

## 问题 2: 模块导入错误

### 问题描述
运行测试时出现 `ModuleNotFoundError: No module named 'toolkits'`

### 错误原因
测试文件位于 `tests/` 子目录，直接导入 `toolkits` 模块会失败，因为 Python 找不到父目录的模块。

### 错误代码
```python
# ❌ 错误：直接导入
from toolkits.browser_tools import get_browser_manager
```

### 修复方案
```python
# ✅ 正确：添加父目录到 Python 路径
import sys
from pathlib import Path

# 添加父目录到 Python 路径
sys.path.insert(0, str(Path(__file__).parent.parent))

from toolkits.browser_tools import get_browser_manager
```
