# 需求文档

## 简介

本文档定义了浏览器代理技能注入系统（Browser Agent Skill Injection System）的需求。该系统支持将特定网站的自动化策略动态注入到浏览器代理中，将通用的浏览器自动化能力与特定网站的知识分离。这使得核心浏览器代理的系统提示词保持精简，专注于通用的浏览器自动化模式，而特定网站的策略（CSS 选择器、操作序列、反爬虫策略）则外部化为可复用的技能文件。

## 术语表

- **Browser_Agent（浏览器代理）**: 负责网页自动化任务的自主代理，定义在 `agent_definition.py` 中
- **Skill（技能）**: 包含特定网站自动化策略、选择器和最佳实践的独立文件
- **Skill_Injection_System（技能注入系统）**: 在运行时加载并将技能内容注入到浏览器代理上下文中的机制
- **System_Prompt（系统提示词）**: 定义浏览器代理行为和能力的核心指令集
- **Skill_Repository（技能仓库）**: 存储技能文件的目录结构（例如 `browser_agent_system_v5/skills/browser/`）
- **Agent_Context（代理上下文）**: 浏览器代理的运行时上下文，包括系统提示词和注入的技能
- **Skill_Metadata（技能元数据）**: 关于技能的结构化信息，包括名称、目标网站、版本和描述

## 需求

### 需求 1：核心系统提示词精简

**用户故事：** 作为系统维护者，我希望浏览器代理的核心系统提示词只包含通用的浏览器自动化指导，以便系统更易于维护和扩展。

#### 验收标准

1. System_Prompt 应当只包含通用的浏览器自动化能力和策略
2. System_Prompt 不应包含特定网站的 CSS 选择器或操作序列
3. System_Prompt 不应包含对特定电商平台（京东、淘宝等）的引用
4. System_Prompt 应当保留所有通用策略，包括 DOM 诊断、基础反爬虫和工作流程模式
5. 当使用重构后的 System_Prompt 时，Browser_Agent 应当保留所有核心自动化能力

### 需求 2：技能文件结构定义

**用户故事：** 作为开发者，我希望有清晰的技能文件格式，以便我可以为不同网站创建新技能。

#### 验收标准

1. Skill_Repository 应当位于 `browser_agent_system_v5/skills/browser/` 目录
2. 每个 Skill 应当作为独立的 Markdown 文件存储，扩展名为 `.md`
3. 每个 Skill 文件应当包含元数据头部分，包括名称、目标网站、版本和描述
4. 每个 Skill 文件应当包含内容部分，包含特定网站的策略和选择器
5. Skill 文件格式应当支持 Markdown 格式以提高可读性
6. 当创建 Skill 文件时，Skill_Injection_System 应当能够解析其元数据和内容
7. 必要时skill中应该包含script，包括但不仅限于爬取网站特定内容所需要的js脚本

### 需求 3：技能发现和加载

**用户故事：** 作为系统，我希望自动发现和加载可用的技能，以便在需要时将它们注入到浏览器代理中。

#### 验收标准

1. 当 Skill_Injection_System 初始化时，系统应当扫描 Skill_Repository 目录
2. 系统应当从每个技能文件中解析 Skill_Metadata
3. 系统应当维护一个按技能名称索引的可用技能内存注册表
4. 当技能文件格式错误时，系统应当记录警告并跳过该技能
5. 系统应当支持热重载技能，无需重启应用程序

### 需求 4：技能注入机制

**用户故事：** 作为浏览器代理，我希望在运行时将相关技能注入到我的上下文中，以便在需要时访问特定网站的知识。

#### 验收标准

1. 当 Browser_Agent 带着任务生成时，Skill_Injection_System 应当确定哪些技能是相关的
2. 系统应当在任务执行前将相关技能内容注入到 Agent_Context 中
3. 注入的技能内容应当作为额外上下文附加到 System_Prompt 后面
4. 系统应当支持同时注入多个技能，用于涉及多个网站的任务
5. 当没有找到相关技能时，Browser_Agent 应当仅使用核心 System_Prompt 执行

### 需求 5：技能选择策略

**用户故事：** 作为系统，我希望根据任务描述智能选择要注入的技能，以便代理拥有相关上下文而不会过载。

#### 验收标准

1. 当任务描述包含 URL 时，系统应当提取域名并与技能目标网站进行匹配
2. 当任务描述包含网站名称（例如"京东"、"JD"、"淘宝"、"Taobao"）时，系统应当与技能元数据进行匹配
3. 系统应当支持使用技能描述进行基于关键词的技能匹配
4. 当多个技能匹配时，系统应当注入所有匹配的技能
5. 系统应当限制注入技能内容的总量以防止上下文溢出（每个技能最多 2000 个 token）

### 需求 6：示例技能迁移 - 京东

**用户故事：** 作为系统维护者，我希望将现有的京东特定策略迁移到技能文件中，以便验证技能系统正常工作。

#### 验收标准

1. 系统应当在 Skill_Repository 中提供名为 `jd_ecommerce.md` 的技能文件
2. 京东技能应当包含直接搜索 URL 模式：`https://search.jd.com/Search?keyword={keyword}&enc=utf-8`
3. 京东技能应当包含处理商品页面异步评论加载的策略
4. 京东技能应当包含评论标签切换（好评/中评/差评）的选择器和技术
5. 当注入京东技能时，Browser_Agent 应当能够访问所有京东特定的自动化知识

### 需求 7：示例技能迁移 - 淘宝

**用户故事：** 作为系统维护者，我希望将现有的淘宝特定策略迁移到技能文件中，以便验证技能系统支持多个平台。

#### 验收标准

1. 系统应当在 Skill_Repository 中提供名为 `taobao_ecommerce.md` 的技能文件
2. 淘宝技能应当包含直接搜索 URL 模式：`https://s.taobao.com/search?q={keyword}`
3. 淘宝技能应当包含任何淘宝特定的反爬虫策略
4. 当注入淘宝技能时，Browser_Agent 应当能够访问所有淘宝特定的自动化知识

### 需求 8：技能注入 API

**用户故事：** 作为开发者，我希望有编程式 API 来控制技能注入，以便我可以自定义技能加载行为。

#### 验收标准

1. 系统应当提供函数 `inject_skills(agent_type: str, task_description: str) -> str`，返回增强的系统提示词
2. 该函数应当接受可选的 `skill_names: List[str]` 参数用于显式技能选择
3. 该函数应当返回组合的系统提示词和注入的技能
4. 当提供显式技能名称时，系统应当仅注入这些技能，无论任务描述如何
5. 如果请求的技能不存在，该函数应当抛出异常

### 需求 9：技能内容验证

**用户故事：** 作为系统，我希望在加载时验证技能文件，以便格式错误的技能不会破坏代理。

#### 验收标准

1. 当加载技能文件时，系统应当验证它包含必需的元数据字段（名称、版本、目标网站）
2. 系统应当验证技能内容部分不为空
3. 系统应当验证技能文件大小不超过 10KB
4. 当验证失败时，系统应当记录详细的错误消息，包括技能文件名和验证失败原因
5. 即使一个技能验证失败，系统也应当继续加载其他技能

### 需求 10：向后兼容性

**用户故事：** 作为系统用户，我希望技能注入系统向后兼容，以便现有的浏览器代理功能继续工作。

#### 验收标准

1. 当 Skill_Repository 中没有可用技能时，Browser_Agent 应当使用核心 System_Prompt 运行
2. 重构后的 System_Prompt 应当保留原始提示词中的所有通用浏览器自动化能力
3. 当 Skill_Injection_System 被禁用时，Browser_Agent 应当回退到核心 System_Prompt
4. 系统不应破坏现有的浏览器代理工具调用（navigate、click_element、extract_text 等）
5. 当运行现有测试用例时，Browser_Agent 应当通过在引入技能系统之前通过的所有测试

### 需求 11：技能版本控制和更新

**用户故事：** 作为开发者，我希望对技能进行版本控制并独立更新它们，以便我可以改进特定网站的策略而不影响核心系统。

#### 验收标准

1. 每个 Skill 应当在其元数据中包含版本字段（语义化版本格式：MAJOR.MINOR.PATCH）
2. 系统应当记录每个注入技能的版本以便调试
3. 当技能文件更新时，系统应当在下次代理生成时重新加载它（热重载）
4. 系统应当在每个技能文件中维护变更日志部分，记录版本变更
5. 当存在多个版本的技能（不同文件名）时，系统应当加载版本号最高的那个

### 需求 12：技能文档和示例

**用户故事：** 作为开发者，我希望有清晰的文档和示例来创建技能，以便我可以轻松添加新的特定网站策略。

#### 验收标准

1. 系统应当在 Skill_Repository 中提供技能模板文件 `_template.md`
2. 模板应当包含所有必需的元数据字段和示例值
3. 模板应当包含示例内容部分，展示最佳实践
4. 系统应当提供文档解释技能文件格式和注入机制
5. 文档应当包含至少两个完整的技能示例（京东和淘宝）
